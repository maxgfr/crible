## Anti-Patterns Verdict

Not AI slop. The "terminal phosphore" identity is real at the token/typography level (own OKLCH palette, one amber accent, phosphor glow reserved for query-bar focus, mono tabular numerals, full light twin). The deterministic detector confirms: **0 findings** at the source tier, zero hex/color bypasses of tokens.css, no px text under 11px. Where it wobbles is component craft: three dropdown paradigms side by side, `window.prompt()` for saving presets, inline `style` attributes — "engineer's scaffold" moments inside a strong identity. Detector caught what the review missed: 2 hardcoded z-indexes bypassing the semantic scale (styles.css:226 `.picker-menu` z:10 = sticky tier; styles.css:582 `.searchbox-results` z:30 = drawer tier). Browser overlay skipped (no browser automation in session).

## Priority Issues

1. **[P1] The URL remembers the company but not the screen.** Query/sort/columns live only in React state; a screen can't be bookmarked or shared — the product's core artifact is unsaveable. Fix: encode `q` (+ sort/columns) in the hash, restore on load. Files: ui/src/router.ts, ui/src/App.tsx.
2. **[P1] Keyboard-dead core loop.** Rows are `<tr onClick>` (unreachable by keyboard — Sam cannot open the drawer from the grid at all), sort is `<th onClick>` with no `aria-sort`, SearchBox is mouse-only despite `role="listbox"`, drawer takes no focus. Fix: focusable rows or symbol links + Enter; `<button>` in `<th>` + aria-sort; real combobox; role=dialog + focus move. Files: ResultsGrid.tsx, SearchBox.tsx, CompanyDrawer.tsx, ColumnPicker.tsx.
3. **[P1] Coverage told as numbers without the why** (observed user pain — "pourquoi on n'a pas tout ?"). The banner states 4,930/151,170 and stops; nothing says audited-filings-only + polite crawl, growing nightly, all listings searchable. `.coverage-banner a` is styled (styles.css:616) but no link is rendered — designed then dropped. Fix: one clause + "why?" link to Status. File: CoverageBanner.tsx.
4. **[P2] Client-side sort over a truncated 500-row page misleads** — sorts an arbitrary slice while meta reads "500 of 8,231". Push sort into the engine (compileSort exists in both clients) and paginate or state truncation. Files: App.tsx:108, ResultsGrid.tsx.
5. **[P2] PresetsMenu doesn't scale at 19** — flat native select, DSL only in option `title`, select resets so active preset invisible, window.prompt save, silent overwrite, no delete. Fix: grouped popover reusing picker-menu vocabulary + mono DSL preview + active-preset display. Files: PresetsMenu.tsx, presets-store.ts.

## Persona Red Flags

- **Alex (power user)**: no shareable/bookmarkable screens; no autocomplete over ~200 fields; mouse-required search results; silent 10k-row export with no busy state; `/` shortcut advertised nowhere; dead-end at row 500.
- **Sam (keyboard/screen reader)**: grid rows unreachable; no aria-sort; drawer without role=dialog/focus trap; listbox semantics announced but not honored (aria-selected hardcoded false); score coloring is color-only in the grid (num-good/num-bad classes) — DESIGN.md promises sign/icon accompaniment, unimplemented. Contrast itself passes AA (tokens deliver).
- **Investor-operator**: StatusView never auto-refreshes (fetch once on mount) and shows no "as of" timestamp; `last_cycle.failed` count is undiagnosable; provenance stops at "provider: yfinance" — never links to the actual ESEF/EDGAR filing; status pill can render "universe ? · no snapshot yet".
