# 0005. Audited US layer: SEC EDGAR companyfacts (keyless, public domain)

- **Status:** accepted (2026-07-13)

## Context
The 2026-07-13 data-sources research (`docs/research/2026-07-13-data-sources/`, 81 cited sources) identified SEC EDGAR as the single biggest keyless opportunity: every US issuer's XBRL facts are public domain, served without a key from `data.sec.gov/api/xbrl/companyfacts/CIK##########.json` (plus a nightly 1.39 GB `companyfacts.zip` bulk). The same research confirmed Yahoo-scraped data is contractually non-redistributable (personal-use ToS) and Google Finance is permanently closed — so EDGAR is the only path to *audited, redistributable* US fundamentals. Separately, implementation review found the audited-outranks-scraped reconciliation (FR-010) silently no-oped: ESEF labels fiscal periods by year ("2024") while yfinance labels them by end date ("2024-12-31"), and `reconcile()` skips audited periods absent from the scraped index — zero snapshot rows carried `audited_fields`.

## Decision
A `provider='edgar'` layer mirroring the ESEF pattern (FR-016): `company_tickers.json` resolves universe tickers to CIKs (dash-form tickers match FinanceDatabase symbols; unmatched are counted, never errored); per-CIK companyfacts JSON is parsed through a conservative us-gaap `CONCEPT_MAP` onto the canonical (yfinance-vocabulary) columns — annual forms only (10-K/20-F/40-F, fp=FY), full-year durations, latest-filed wins, capex sign negated to the yfinance convention; frames land in the immutable raw layer and outrank scraped values at reconciliation. A new `align_periods` step maps audited period labels onto the scraped label of the same fiscal year, fixing the dormant ESEF reconciliation for free. SEC fair-access compliance: a declared User-Agent (`CRIBLE_SEC_USER_AGENT`, default names the repo) and a dedicated 5 req/s token bucket — never the Yahoo budget. The `edgar_tasks` cycle refreshes each symbol every 90 days.

## Consequences
US symbols in the crawl (20 mega-caps in the bootstrap sample, all US listings at scale) gain audited statements; discrepancies vs scraped values are logged with provenance. EDGAR accepts datacenter IPs (unlike Yahoo), so the nightly GitHub Actions refresh enriches US symbols reliably. Prices still come from yfinance — no redistributable OHLCV source exists anywhere (exchanges monetize it) — so valuation ratios remain scrape-dependent. **Update (2026-07-13): the 1.39 GB `companyfacts.zip` bulk path is implemented** (`run_edgar_bulk`, `refresh --edgar-bulk`, on in the nightly): every resolved US listing (~10k issuers) gets its audited layer in one sweep, history bounded to 8 fiscal years to keep the published snapshot under GitHub's file limits. Priceless issuers carry price-free scores (Piotroski, Beneish, margins, growth) with NULL valuation ratios — never imputed.

## Alternatives considered
edgartools as a dependency — rejected: a heavy library for the narrow slice needed; the thin ESEF-style client is ~150 lines and fully fixture-testable. Bulk `companyfacts.zip` now — deferred: 1.39 GB per nightly run outweighs the ~100-symbol demo need. SEC quarterly Financial Statement Data Sets — deferred: quarterly cadence and heavier parsing for no additional benefit at this scale.
