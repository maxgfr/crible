# 0007. The hosted site is the product; publish the OHLCV price series

- **Status:** accepted

## Context
The GitHub Pages build was framed as a "demo" — a limited preview of the self-hosted screener — but it is not limited: it runs the same DSL (golden-locked TS port), the same DuckDB queries and the same CSV export as the API, over the same published dataset. The "demo" label was naming, a UI banner, and one real data exclusion: price **series** were withheld. `crible import-prices` distilled every dump to one derived row per symbol (last close, as-of, 6-month return) and the crawled yfinance OHLCV bars, though present in the raw layer on the published branch/release, were never exported to the browser. The exclusion was a licensing stance (Yahoo/HuggingFace/Stooq series are not open data), not a size constraint — the full 1-year series for the crawled sample is ~1.8 MB, orders of magnitude under GitHub's 100 MB file / 1 GB Pages / 2 GB release limits.

## Decision
Drop the "demo" concept everywhere: the hosted site is the product, the published dataset is the dataset. Concretely:
- Rename the distribution branch `demo-data` → `data`; rename `crible demo-refresh` → `crible refresh`, the publish/seed scripts, the UI `DemoBanner` → `CoverageBanner` (a neutral coverage line, no upsell), and the `DemoManifest` type → `SiteManifest`.
- Publish the daily OHLCV **series**, accepting the redistribution risk explicitly. `import-prices` now persists the windowed series (last 400 days) to `data/prices/<source>.parquet` in addition to the distillate; `crible export-site` assembles every symbol's series (crawled yfinance bars + imported dumps, one whole source winning per symbol — bases never mixed) into symbol-sorted, size-bounded Parquet shards `site-data/prices-*.parquet`, listed in `manifest.json` (`schema: 2`, a `prices` block). The UI serves a per-symbol daily close chart in the company drawer via a new `prices(symbol)` seam (DuckDB range-query in static mode, `/api/prices/{symbol}` in API mode).
- Keep the guards that are not "demo" logic: the `export-site --min-symbols` floor (never publish an empty/undersized dataset), the nightly bounds (`--deadline`, ESEF/EDGAR caps, token-bucket budget), and the distillate (`prices-latest.parquet`) as the snapshot's price fallback for EDGAR-only issuers.

## Consequences
The browser now shows real price history, and the published dataset is the full publishable set. The redistribution risk on the price series is assumed and dated here (2026-07-13) — the same standing caveat that already applied to the scraped fundamentals sample, now extended to the series. Size is bounded defensively: each shard is asserted under 95 MB (below the 100 MB git wall) and the total is watched via the manifest `prices.bytes`; the only scenario that approaches the Pages 1 GB ceiling is a full-universe daily series (~2.8 GB), which the current ~100-symbol yfinance crawl and universe-filtered dumps are far from. A shrinking shard count can leave a stale `prices-NN.parquet` release asset (`--clobber` replaces, never deletes) — the tarball and manifest stay authoritative.

## Alternatives considered
Distillate-only (the ADR-0006 status quo) — rejected: it withheld real, publishable data behind a naming distinction and gave the browser no price history. Publishing series as one unbounded file — rejected: a Stooq worldwide import can leap past the 100 MB git wall in a single nightly; sharding is required. Per-region shards — rejected: uneven and drifting, and region is not a column on a price row; symbol-sorted disjoint shards let DuckDB prune on zone maps and leave the door open to registering only the covering shard later. Adding a chart library — rejected: a ~60-line hand-rolled SVG matches the existing no-chart-lib style (StatusView) and adds no dependency.
